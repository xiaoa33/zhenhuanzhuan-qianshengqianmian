# 甄嬛传 14 角色语音数据集说明

> ⚠️ **早期构建过程中曾候选 17 角色**（14 + 阿晋 + 净白 + 齐妃），经人工标注三轮验证后淘汰 3 个不可靠簇，最终保留 14 个高可信度角色。文档中部分统计值为中间版本快照，最终数据为 13,605 段。

## 概述

- **数据来源**：《甄嬛传》76 集 MP3 音频（无人播剧版，去片头片尾，128kbps, 44.1kHz stereo）
- **角色数量**：14 人
- **总片段数**：~13,605 段（清洗前 14,315 段）
- **总时长**：约 1,005 分钟（16.7 小时）

> 已剔除 3 个角色：阿晋、净白师太、齐妃。原因是 KMeans 聚类后人工试听确认这三个 spk cluster 内混入了多个不同说话人的片段，音色杂乱不可靠。
>
> 早期候选为 17 角色（14 + 阿晋 + 净白 + 齐妃），经人工标注三轮验证后淘汰不可靠的 3 个簇，最终保留 14 个高可信度角色。

## 文件格式详解

### wav.scp

每行：`utt_id` + 空格 + `音频绝对路径`

```
zhenhuan_0001 /root/autodl-tmp/data/segments/zhz_01_seg0311.wav
zhenhuan_0002 /root/autodl-tmp/data/segments/zhz_24_seg0290.wav
huafei_0001 /root/autodl-tmp/data/segments/zhz_20_seg0127.wav
huangshang_0001 /root/autodl-tmp/data/segments/zhz_09_seg0162.wav
...
```

> 注意：路径是绝对路径。迁移到新服务器后需要用 `sed` 批量替换为实际路径。

### text

每行：`utt_id` + 空格 + `文本内容`

```
zhenhuan_0001 从来英雄不问出身妹妹美貌又何必妄自菲薄
huafei_0001 贱人就是矫情
huangshang_0001 朕与皇后要去天坛主持祭祀之礼
```

> 注意：部分片段文本被 FunASR 输出为逐字空格分隔格式（如 `"臣 妾 不 敢"`），下游训练时可能需要预处理去除空格。

### utt2spk

每行：`utt_id` + 空格 + `角色名`

```
zhenhuan_0001 甄嬛
huafei_0001 华妃·年世兰
huangshang_0001 皇上·爱新觉罗·胤禎
huanghou_0001 乌拉那拉·宜修(皇后)
```

### spk2utt

每行：`角色名` + 空格 + `utt_id 列表（空格分隔）`

```
甄嬛 zhenhuan_0001 zhenhuan_0002 zhenhuan_0003 ...
华妃·年世兰 huafei_0001 huafei_0002 huafei_0003 ...
```

### instruct（可选）

每行：`utt_id` + 空格 + `语气指令文本`

```
huafei_0001 用华妃的语气说话，嚣张跋扈，目中无人，语气凌厉。
zhenhuan_0001 用甄嬛的语气说话，温柔但坚定，端庄大气。
```

## 数据处理流程摘要

```
原始 76 集 MP3（44.1kHz stereo）
    ↓ demucs (mdx_extra_q) 人声分离
纯净人声（44.1kHz stereo）
    ↓ FunASR VAD 切分 + ASR 转写
短片段 WAV（16kHz mono）+ 空格分隔文本
    ↓ 时间戳合并（间隔 > 800ms 切分）
    ↓ eres2netv2 speaker embedding + KMeans 聚类
    ↓ 人工试听标注（14 角色）
    ↓ 文本去重 + 音频质检（RMS / 削波）
13,605 段最终数据（清洗前 14,315 段）
```

## 适配其他模型

这个数据集是通用的 **音频 + 文本 + 说话人标签** 格式，不仅限于 CosyVoice。

### 适配 GPT-SoVITS

GPT-SoVITS 需要每个角色一个子目录 + 文本列表：

```bash
# 按角色拆分为 GPT-SoVITS 格式
while read spk utts; do
  mkdir -p gpt_sovits_data/$spk
  for utt in $utts; do
    wav=$(grep "^$utt " wav.scp | cut -d' ' -f2)
    text=$(grep "^$utt " text | cut -d' ' -f2-)
    cp "$wav" gpt_sovits_data/$spk/
    echo "$utt|$spk|ZH|$text" >> gpt_sovits_data/$spk/filelist.txt
  done
done < spk2utt
```

### 适配 VITS / Bert-VITS2

直接使用 `wav.scp` + `text` + `utt2spk` 即可，格式与 Kaldi/ESPnet 兼容。

### 适配 Fish-Speech

按角色分目录存放 WAV + 同名的 `.lab` 文本标注文件。

## 注意事项

1. **路径是绝对路径**：`wav.scp` 中的音频路径指向生成服务器的目录。换服务器后需用 `sed` 替换
2. **文本有空格分隔版**：部分片段由 FunASR 输出为逐字空格分隔，建议训练前统一预处理
3. **少量片段含多说话人**：VAD 可能未切分紧邻对话（< 5%），一般不影响训练
4. **已去重**：完全相同文本只保留一条
5. **已质检**：过滤了静音（RMS < 0.003）和削波片段（> 10%）
6. **可复现**：`speaker_map.json` 保存了完整映射表，原始 MP3 和处理脚本均在仓库中
