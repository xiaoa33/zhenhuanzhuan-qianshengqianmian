# 甄嬛传 AI 角色语音对话系统 — 数据集构建流程

## 项目概述

基于《甄嬛传》全集音频，构建**每个主要角色的干净语音片段 + 文本标注**数据集，供 CosyVoice 语音克隆模型训练使用。

> **运行环境**：全程在云端 GPU 服务器上运行（单卡 RTX 3090 即可），本地无需 GPU。
> 原始音频 76 集 MP3 上传云端后，一键脚本 `scripts/run_all.sh` 串联全流程。
> 自动化处理脚本位于 `scripts/` 目录，详见 [scripts/README.md](../scripts/README.md)。

## 2. 整体 Pipeline

```
┌─────────────────────────────────────────────────────────────────┐
│                  甄嬛传全集 MP3 / WAV                            │
└────────────────────────┬────────────────────────────────────────┘
                         │
          ┌──────────────▼──────────────┐
          │  Step 1: 人声分离 (demucs)   │  去掉背景音乐、环境音
          │  → 输出: 纯净人声            │
          └──────────────┬──────────────┘
                         │
          ┌──────────────▼──────────────┐
          │  Step 2: VAD + ASR + 说话人 │  FunASR Paraformer + CAM++
          │  → 输出: 片段+文本+speakerID │  (step1_pipeline.py 三合一)
          └──────────────┬──────────────┘
                         │
          ┌──────────────▼──────────────┐
          │  Step 3: 角色标注            │  speaker_id → 角色名
          │  → 输出: 角色名 + 音频 + 文本 │  (step2_sample_labeling.py)
          └──────────────┬──────────────┘
                         │
          ┌──────────────▼──────────────┐
          │  Step 4: 数据筛选 & 清洗     │  去噪、去重、质量控制
          │  → 输出: 每个角色 ≥ 10min    │  (step3_clean.py)
          └──────────────┬──────────────┘
                         │
          ┌──────────────▼──────────────┐
          │  Step 5: 组织训练集格式      │  Kaldi + Embedding/Token
          │  → 输出: 最终训练数据         │  (step4 + step5)
          └─────────────────────────────┘
```

## 3. 各步骤详细说明

### Step 1: 人声分离（demucs）

目的：去掉背景音乐、环境音效，提取纯净人声。

使用 Meta 开源的 **demucs** 模型（~80MB，轻量可靠），将每集 MP3 分离为人声 + 背景两轨：

```bash
# 安装
pip install demucs

# 试跑第 1 集，验证效果
python -m demucs --two-stems=vocals \
  -o data/clean_vocals \
  "data/甄嬛传原声_无人播剧音频去片头片尾_正常音量_MP3/zhz_01.mp3"

# 确认人声干净后，批量处理全部 76 集
export HF_ENDPOINT=https://hf-mirror.com  # 国内镜像加速首次模型下载
for f in data/甄嬛传原声_无人播剧音频去片头片尾_正常音量_MP3/zhz_*.mp3; do
  python -m demucs --two-stems=vocals -o data/clean_vocals "$f"
done
```

输入 / 输出：

```
raw_audio/
├── zhz_01.mp3
├── zhz_02.mp3
└── ...

clean_vocals/htdemucs/
├── zhz_01/
│   ├── vocals.wav       ← 纯净人声（保留）
│   └── no_vocals.wav    ← 背景音乐（丢弃）
├── zhz_02/
│   ├── vocals.wav
│   └── no_vocals.wav
└── ...
```

> **替代方案**：如果 demucs 效果不理想，可改用 UVR5 GUI（[github.com/Anjok07/ultimatevocalremovergui](https://github.com/Anjok07/ultimatevocalremovergui)），模型选 `MDX-Net → Kim_Vocal_2`。

### Step 2: VAD 切分 + ASR 转写 + 说话人识别（三合一）

目的：一次推理完成语音切分、文字转写、说话人聚类。

使用 FunASR 的 SeaCo-Paraformer 模型（同时支持 VAD + ASR + Speaker Diarization），
由 `scripts/step1_pipeline.py` 一键完成：

```bash
python scripts/step1_pipeline.py
```

核心模型配置（详见 [config.py](../scripts/config.py)）：
- ASR: `iic/speech_seaco_paraformer_large_asr_nat-zh-cn-16k-common-vocab8404-pytorch`
- VAD: `iic/speech_fsmn_vad_zh-cn-16k-common`
- 标点: `iic/punc_ct-transformer_zh-cn-common-vad_realtime-vocab272727`
- 说话人: `iic/speech_campplus_sv_zh-cn_16k-common`

输入 / 输出：

```
clean_vocals/htdemucs/zhz_01/vocals.wav  (~40分钟)
    ↓  step1_pipeline.py
data/segments/                           (音频片段)
├── zhz_01_seg0000.wav                   (3.2秒)
├── zhz_01_seg0001.wav                   (5.1秒)
└── ...                                  (~500-800段/集)

data/pipeline_results/all_segments.json  (完整标注)
[
  {"file": "zhz_01_seg0000.wav", "text": "臣妾不敢妄议朝政。", "speaker": "spk_01", "duration": 3.2},
  {"file": "zhz_01_seg0001.wav", "text": "这后宫之中...",      "speaker": "spk_03", "duration": 5.1},
  ...
]
```

> 详细实现见 `scripts/step1_pipeline.py`。
  ...
]
```

### Step 3: 角色标注（speaker_id → 角色名）

目的：把机器给出的 `spk_01` 映射到真正的角色名。

使用 `scripts/step2_sample_labeling.py` 辅助完成：

```bash
# 第 1 步：为每个 speaker 随机抽样 + 关键词自动提示
python scripts/step2_sample_labeling.py

# 第 2 步：人工试听样本，编辑 data/pipeline_results/speaker_map.json
#     格式: {"spk_01": "甄嬛", "spk_02": "华妃", ...}
#     无法确定的标为 ""（将被过滤为 "other"）

# 第 3 步：应用映射表
python scripts/step2_sample_labeling.py --apply-map
```

核心逻辑：
1. 每个 speaker 随机抽 10 条 → 人工听音频 + 看文本 → 判断角色
2. 内置关键词匹配提示（如"朕"→皇上，"贱人"→华妃），可加速标注
3. 目标角色列表在 [config.py](../scripts/config.py) 的 `TARGET_CHARACTERS` 中维护

> **加速技巧**：如果手头有《甄嬛传》完整剧本，可以用剧本文本跟 ASR 结果做**模糊匹配**来自动标注角色，准确率能到 80%，剩下的 20% 再人工校对。

---

### Step 4: 数据筛选 & 清洗

目的：去掉低质量片段，确保每个角色有足够的高质量训练数据。

```bash
python scripts/step3_clean.py
```

筛选规则（详见 [config.py](../scripts/config.py)）：
1. 只保留 12 个目标角色的片段
2. 文本长度 2~100 字
3. 音频时长 2~15 秒
4. RMS 能量 > 0.003（过滤静音/极轻片段）
5. 去除完全重复的文本
6. 削波比例 < 10%

**质量标准参考**：

| 指标 | 最低要求 | 推荐 |
|------|----------|------|
| 每角色音频总量 | ≥ 10分钟 | ≥ 30分钟 |
| 单段时长 | 2~15秒 | 3~10秒 |
| 采样率 | 16kHz | 22050Hz (CosyVoice 默认) |
| 格式 | WAV, mono | WAV, mono, 22050Hz |
| 背景噪声 | 可接受少量 | 尽可能干净 |

### Step 5: 组织 CosyVoice 训练格式

CosyVoice 使用 **Kaldi 风格文本文件 + Parquet** 的数据格式（不是常见的 CSV/JSON），训练流程分为三个阶段。

#### 7.1 第一阶段：生成 Kaldi 风格文本文件

```
data/train/
├── wav.scp              # utt_id → 音频路径
├── text                 # utt_id → 文本内容
├── utt2spk             # utt_id → speaker_id
├── spk2utt             # speaker_id → utt_id列表
└── instruct            # (可选) utt_id → 指令文本
```

**wav.scp** 格式（`utt_id 音频路径`）：
```
zhenhuan_0001 data/segments/zhenhuan/zhenhuan_0001.wav
zhenhuan_0002 data/segments/zhenhuan/zhenhuan_0002.wav
huafei_0001  data/segments/huafei/huafei_0001.wav
...
```

**text** 格式（`utt_id 文本`）：
```
zhenhuan_0001 臣妾不敢妄议朝政。
zhenhuan_0002 这后宫之中谁人不知华妃娘娘的厉害。
huafei_0001 贱人就是矫情！
...
```

**utt2spk** 格式（`utt_id speaker_id`）：
```
zhenhuan_0001 zhenhuan
zhenhuan_0002 zhenhuan
huafei_0001 huafei
...
```

**spk2utt** 格式（`speaker_id utt_id1 utt_id2 ...`）：
```
zhenhuan zhenhuan_0001 zhenhuan_0002 zhenhuan_0003 ...
huafei huafei_0001 huafei_0002 huafei_0003 ...
...
```

**instruct**（可选，`utt_id 指令文本`）：
```
zhenhuan_0001 用甄嬛的语气说话，端庄大气，不卑不亢。
huafei_0001 用华妃的语气说话，嚣张跋扈，目中无人。
```

```python
# build_kaldi_format.py
import json
import os
import shutil
from collections import defaultdict

def build_kaldi_files(clean_data_json, output_dir="data/train/"):
    """
    从清洗后的标注数据构建 CosyVoice 所需的 Kaldi 风格文件
    """
    with open(clean_data_json, encoding="utf-8") as f:
        data = json.load(f)

    os.makedirs(output_dir, exist_ok=True)

    # 按角色分组
    by_character = defaultdict(list)
    for item in data:
        by_character[item["character"]].append(item)

    # 按 speaker 存放音频到 segments/ 下
    seg_dir = os.path.join(output_dir, "segments")
    wav_scp_lines = []
    text_lines = []
    utt2spk_lines = []
    spk2utt = defaultdict(list)

    for character, items in by_character.items():
        char_seg_dir = os.path.join(seg_dir, character)
        os.makedirs(char_seg_dir, exist_ok=True)

        for i, item in enumerate(items):
            utt_id = f"{character}_{i+1:04d}"

            # 复制音频
            dst_name = f"{utt_id}.wav"
            dst_path = os.path.join(char_seg_dir, dst_name)
            src_path = os.path.join("segments", item["file"])
            if os.path.exists(src_path):
                shutil.copy(src_path, dst_path)

            # 写入各文件内容
            wav_scp_lines.append(f"{utt_id} {dst_path}")
            text_lines.append(f"{utt_id} {item['text']}")
            utt2spk_lines.append(f"{utt_id} {character}")
            spk2utt[character].append(utt_id)

    # 写入 wav.scp
    with open(os.path.join(output_dir, "wav.scp"), "w", encoding="utf-8") as f:
        f.write("\n".join(wav_scp_lines))

    # 写入 text
    with open(os.path.join(output_dir, "text"), "w", encoding="utf-8") as f:
        f.write("\n".join(text_lines))

    # 写入 utt2spk
    with open(os.path.join(output_dir, "utt2spk"), "w", encoding="utf-8") as f:
        f.write("\n".join(utt2spk_lines))

    # 写入 spk2utt
    with open(os.path.join(output_dir, "spk2utt"), "w", encoding="utf-8") as f:
        for spk, utts in spk2utt.items():
            f.write(f"{spk} {' '.join(utts)}\n")

    # 统计
    print(f"Kaldi 格式文件已生成到: {output_dir}")
    for spk, utts in sorted(spk2utt.items()):
        total_dur = sum(it["duration"] for it in by_character[spk])
        print(f"  {spk}: {len(utts)}段, {total_dur/60:.1f}分钟")

    return spk2utt

build_kaldi_files("clean_data.json", "data/train/")
```

#### 7.2 第二阶段：提取 Speaker Embedding 和 Speech Token

这是 CosyVoice 微调必需的步骤，需要用到预训练模型提供的 ONNX 文件。

```bash
# 1. 提取 speaker embedding（CAM++ 模型）
#    生成 data/train/spk2embedding.pt 和 data/train/utt2embedding.pt
python CosyVoice/tools/extract_embedding.py \
    --dir data/train/ \
    --onnx_path pretrained_models/CosyVoice-300M/campplus.onnx

# 2. 提取 speech token（用于 LLM 训练）
#    生成 data/train/utt2speech_token.pt
python CosyVoice/tools/extract_speech_token.py \
    --dir data/train/ \
    --onnx_path pretrained_models/CosyVoice-300M/speech_tokenizer_v1.onnx
```

#### 7.3 第三阶段：转换为 Parquet 格式（最终训练格式）

```bash
# 将所有文本文件 + embedding + token 打包为 Parquet
# 输出 data/train/parquet/data.list
python CosyVoice/tools/make_parquet_list.py \
    --num_utts_per_parquet 1000 \
    --num_processes 4 \
    --src_dir data/train/ \
    --des_dir data/train/parquet/
```

执行后得到最终训练数据：
```
data/train/parquet/
├── parquet_000000000.tar    # 每个包含 1000 条数据
├── parquet_000000001.tar    # 列: utt, audio_data(原始字节), wav, text,
├── ...                      #      spk, spk_embedding, utt_embedding, speech_token
├── data.list                # 所有 parquet 文件路径列表
├── utt2data.list
└── spk2data.list
```

> **注意**：CosyVoice 训练时默认将音频重采样到 **22050Hz**（在 processor pipeline 的 `resample` 阶段自动完成），因此前面的片段无需手动重采样。

## 5. CosyVoice 训练使用

### 5.1 零样本推理（无需训练，3秒参考音频即可克隆）

```python
from cosyvoice.cli.cosyvoice import CosyVoice

model = CosyVoice('iic/CosyVoice-300M')
output = model.inference_zero_shot(
    "贱人就是矫情！",
    "你看不惯本宫，本宫还看不惯你呢！",
    prompt_wav="./huafei/huafei_0001.wav"  # 参考音频
)
```

### 5.2 微调训练（使用构建的数据集）

构建完 `data/train/parquet/` 后，使用 CosyVoice 的训练脚本进行微调：

```bash
# 合并训练集和验证集
cat data/train/parquet/data.list > data/train.data.list
cat data/dev/parquet/data.list > data/dev.data.list

# 设置 GPU
export CUDA_VISIBLE_DEVICES="0,1,2,3"

# 依次训练三个模型：LLM → Flow → HiFiGAN
# 使用预训练模型作为起点进行微调（SFT）
pretrained_dir=pretrained_models/CosyVoice-300M

for model in llm flow hifigan; do
  torchrun --nnodes=1 --nproc_per_node=4 \
      --rdzv_id=1234 --rdzv_backend="c10d" --rdzv_endpoint="localhost:1234" \
    CosyVoice/cosyvoice/bin/train.py \
      --config conf/cosyvoice.yaml \
      --train_data data/train.data.list \
      --cv_data data/dev.data.list \
      --model $model \
      --checkpoint $pretrained_dir/$model.pt \
      --model_dir exp/cosyvoice/$model \
      --tensorboard_dir tensorboard/cosyvoice/$model \
      --num_workers 2 \
      --prefetch 100 \
      --pin_memory \
      --use_amp
done
```

> **微调注意事项**：
> - 微调时建议降低学习率：LLM `lr: 1e-5`，Flow `lr: 1e-5`（配置文件中已注释）
> - 使用 `use_spk_embedding: True` 进行 SFT（在 `padding` 配置中）
> - 微调后需运行 `average_model.py` 和 `export_jit.py`/`export_onnx.py` 导出模型
> - 如果使用 CosyVoice-300M-25Hz 版本（中文推荐），需修改 tokenizer 为 `cosyvoice.tokenizer.tokenizer.get_tokenizer`，并将 `input_frame_rate` 改为 25

### 5.3 模型导出

```bash
# 平均最后5个checkpoint
python CosyVoice/cosyvoice/bin/average_model.py \
    --dst_model exp/cosyvoice/llm/llm.pt \
    --src_path exp/cosyvoice/llm/ \
    --num 5 --val_best

# 导出加速
python CosyVoice/cosyvoice/bin/export_jit.py --model_dir exp/cosyvoice/
python CosyVoice/cosyvoice/bin/export_onnx.py --model_dir exp/cosyvoice/
```

**CosyVoice 相比 GPT-SoVITS 的优势**：

| 特性 | CosyVoice | GPT-SoVITS |
|------|-----------|------------|
| 零样本克隆 | ✅ 3秒音频即可 | ❌ 需要微调 |
| 情感控制 | ✅ 内置 | ⚠️ 需额外处理 |
| 中文效果 | ✅ 阿里训练，极好 | ✅ 好 |
| 训练数据量 | 零样本无需训练；微调≥10分钟 | ≥30分钟 |
| 推理速度 | 快（流式） | 中等 |
| 指令控制 | ✅ 支持 instruct | ❌ 不支持 |

## 7. 常见问题

**Q: 角色标注太慢怎么办？**
A: 三种加速方式：
1. 用剧本台词文本做关键词匹配自动标注（如"朕"、"皇上"大概率是皇上说的）
2. 只处理主角戏份多的集数（不必处理全部76集）
3. 训练一个小分类器：手工标1000条 → 训练 → 预测剩余 → 抽样校对

**Q: CosyVoice 零样本已经很好了，为什么还要微调？**
A: 零样本3秒克隆语气和音色稳定性有限。微调后：
- 音色更稳定、更像
- 能学到角色的说话习惯（如华妃的抑扬顿挫）
- 支持情感控制

**Q: 每个角色最少需要多少数据？**
A: CosyVoice 微调 ≥ 10分钟即可出效果，≥ 30分钟效果稳定。优选主角：
- 甄嬛：戏份最多，预计可提取 60+ 分钟
- 华妃：前40集核心反派，预计 40+ 分钟
- 皇上：全程出场，预计 50+ 分钟
