# 三人异地联调说明

本文档记录当前项目在“三个人不共用一台电脑”的情况下如何联调。

## 1. 当前分工

| 成员 | 机器/环境 | 负责模块 |
|---|---|---|
| 你 | 校内服务器 | `yiping-backend`、前端、`xiao-asr_llm`、GPT-SoVITS |
| CosyVoice 队友 | 算力自由服务器/她自己的机器 | CosyVoice TTS |
| 其他队友 | 自己电脑 | 通过浏览器访问网站、测试功能 |

关键约束：

- GPT-SoVITS 权重在你的校内服务器上，所以 GPT-SoVITS 服务必须由你启动。
- CosyVoice 模型在队友机器上，所以 CosyVoice 服务由队友启动。
- 主后端 `yiping-backend` 推荐由你启动，作为统一入口，负责分发到 GPT-SoVITS 或 CosyVoice。

## 2. 推荐整体架构

```text
用户/队友浏览器
  -> 你的校内服务器前端 :5173
      -> 你的校内服务器 yiping-backend :8003
          -> 本机 xiao-asr_llm :8001
          -> 本机 GPT-SoVITS service :8004
          -> CosyVoice 队友服务器 :8002 或隧道端口
```

不要让队友直接调用 GPT-SoVITS 的 `8004`。GPT-SoVITS 服务返回的是服务器本地 `audio_path`，远程电脑不能直接读取。队友应该通过 `yiping-backend :8003` 调用，主后端会复制音频并返回可访问的 `audio_url`。

## 3. 如何查看你的校内服务器可访问地址

在你的校内服务器上执行：

```bash
hostname -I
```

或：

```bash
ip addr
```

常见校内网 IP 形如：

```text
10.x.x.x
172.x.x.x
192.168.x.x
```

假设查到：

```text
10.12.34.56
```

后续文档中的 `你的校内IP` 就替换为：

```text
10.12.34.56
```

注意：查到 IP 不代表外部一定可访问。最终以队友电脑能否访问为准。

## 4. 你的服务启动方式

### 4.1 启动 xiao-asr_llm

```bash
cd xiao-asr_llm
uvicorn main:app --host 0.0.0.0 --port 8001
```

如果只给本机主后端访问，`--host 127.0.0.1` 也可以。但为了统一，联调时可用 `0.0.0.0`。

### 4.2 启动 GPT-SoVITS

GPT-SoVITS 现在约定使用 `8004`，避免占用 CosyVoice 原来的 `8002`。

推荐常驻缓存模式：

```bash
cd xiao-gpt-sovits

GSV_BACKEND=persistent GSV_CACHE_SIZE=1 \
  python -m uvicorn main:app --host 0.0.0.0 --port 8004
```

第一次请求某个角色会加载模型，较慢；同一角色第二次开始会复用缓存。

### 4.3 启动 yiping-backend

```bash
cd yiping-backend
uvicorn main:app --host 0.0.0.0 --port 8003
```

### 4.4 启动前端

```bash
cd yiping-frontend
npm run dev -- --host 0.0.0.0
```

## 5. 你的 yiping-backend/.env 配置

如果你的校内 IP 是 `10.12.34.56`，推荐：

```env
USE_MOCK=true
USE_MOCK_TTS=false
USE_MOCK_LLM=false

LLM_SERVICE_URL=http://localhost:8001

TTS_SERVICE_URL=http://localhost:8002
GPT_SOVITS_SERVICE_URL=http://localhost:8004
COSYVOICE_SERVICE_URL=http://队友CosyVoice可访问地址:8002

STATIC_BASE_URL=http://10.12.34.56:8003
CORS_ORIGINS=http://10.12.34.56:5173

PORT=8003
```

说明：

- `LLM_SERVICE_URL` 用 `localhost`，因为 `xiao-asr_llm` 和主后端都在你服务器上。
- `GPT_SOVITS_SERVICE_URL` 用 `localhost:8004`，因为 GPT-SoVITS 和主后端都在你服务器上。
- `COSYVOICE_SERVICE_URL` 要填你服务器能访问到的 CosyVoice 地址。
- `STATIC_BASE_URL` 必须是队友浏览器能访问的地址，不能写 `localhost`。
- `CORS_ORIGINS` 是允许访问主后端的前端地址。

临时联调时，如果 CORS 一直报错，可以暂时设：

```env
CORS_ORIGINS=*
```

正式展示或公开部署不建议长期使用 `*`。

## 6. 前端 .env.local 配置

文件：

```text
yiping-frontend/.env.local
```

如果你的校内 IP 是 `10.12.34.56`：

```env
VITE_API_BASE_URL=http://10.12.34.56:8003
VITE_TTS_ENGINE=gpt_sovits
```

说明：

- `VITE_API_BASE_URL` 是浏览器访问主后端的地址。
- `VITE_TTS_ENGINE` 只是默认选项；前端页面内仍可在 `GPT-SoVITS` 和 `CosyVoice` 之间切换。

修改 `.env.local` 后必须重启前端。

## 7. CosyVoice 队友如何接入

CosyVoice 队友需要提供一个你的校内服务器能访问的地址。

### 方案 A：CosyVoice 服务器本身有可访问地址

如果队友的算力自由服务器可从你的校内服务器访问，例如：

```text
http://cosyvoice-server.example.com:8002
```

你的 `.env` 写：

```env
COSYVOICE_SERVICE_URL=http://cosyvoice-server.example.com:8002
```

### 方案 B：队友通过 SSH 反向隧道接入你的服务器

如果队友的 CosyVoice 服务只在她本机/服务器本地 `8002` 可访问，可以让她在她的机器上执行：

```bash
ssh -N -R 18002:localhost:8002 user@你的校内服务器
```

然后你服务器上的 `.env` 写：

```env
COSYVOICE_SERVICE_URL=http://localhost:18002
```

含义：你的服务器访问 `localhost:18002` 时，会通过 SSH 反向隧道转发到队友机器的 `localhost:8002`。

### 方案 C：CosyVoice 在云端 AutoDL

如果 CosyVoice 云端服务实际跑在云端 `8003`，你可以在你的校内服务器上建本地隧道：

```bash
ssh -L 8002:localhost:8003 -p 28281 root@connect.bjb1.seetacloud.com
```

你的 `.env` 写：

```env
COSYVOICE_SERVICE_URL=http://localhost:8002
```

## 8. VPN 联调

如果你的校内服务器没有公网入口，但学校有 VPN APP，队友可以尝试：

1. 队友在自己电脑登录学校 VPN APP。
2. 你启动后端和前端时使用 `--host 0.0.0.0`。
3. 队友测试：

```bash
curl http://你的校内IP:8003/
```

如果返回：

```json
{"status":"ok", "...": "..."}
```

说明队友通过 VPN 可以访问你的主后端。

再测试前端：

```text
http://你的校内IP:5173
```

如果 VPN 后仍不能访问，常见原因：

- VPN 只允许访问学校网页系统，不允许访问实验室服务器网段。
- 服务器防火墙没开放 `5173` / `8003`。
- 服务启动时没有加 `--host 0.0.0.0`。
- 你拿到的是容器内部 IP，不是校内可路由 IP。
- 学校 VPN 账号没有访问该服务器网段的权限。

## 9. CORS_ORIGINS 是什么

`CORS_ORIGINS` 是浏览器跨域访问白名单。

浏览器认为下面两个地址是不同源：

```text
http://10.12.34.56:5173
http://10.12.34.56:8003
```

因为端口不同。前端从 `5173` 请求后端 `8003` 时，后端必须明确允许这个前端地址。

所以：

```env
CORS_ORIGINS=http://10.12.34.56:5173
```

如果有多个前端地址，用逗号分隔：

```env
CORS_ORIGINS=http://10.12.34.56:5173,http://127.0.0.1:5173
```

临时联调：

```env
CORS_ORIGINS=*
```

注意：CORS 只能解决浏览器安全策略问题，不能解决网络不通。如果队友电脑本身访问不到你的服务器 IP，改 CORS 没用。

## 10. 验证命令

### 10.1 你在服务器本机验证

```bash
curl http://localhost:8001/
curl http://localhost:8003/
curl http://localhost:8004/
```

应该分别看到：

- `xiao-asr_llm` 状态
- `yiping-backend` 状态
- GPT-SoVITS service 状态

### 10.2 队友验证主后端

队友电脑上：

```bash
curl http://你的校内IP:8003/
```

### 10.3 通过主后端测试 GPT-SoVITS

```bash
curl -X POST http://你的校内IP:8003/synthesize \
  -H "Content-Type: application/json" \
  -d '{
    "character_id": "zhenhuan",
    "text": "本宫今日心情尚可，你倒是有眼力见儿。",
    "emotion": "喜悦",
    "engine": "gpt_sovits"
  }'
```

成功时返回：

```json
{
  "audio_url": "http://你的校内IP:8003/static/audio/reply_xxx.wav"
}
```

### 10.4 通过主后端测试 CosyVoice

```bash
curl -X POST http://你的校内IP:8003/synthesize \
  -H "Content-Type: application/json" \
  -d '{
    "character_id": "zhenhuan",
    "text": "[laughter]本宫今日心情尚可，你倒是有<strong>眼力见儿</strong>。",
    "emotion": "喜悦",
    "engine": "cosyvoice"
  }'
```

成功时同样返回 `audio_url`。

## 11. 推荐联调顺序

1. 你先在服务器本机确认 `8001`、`8003`、`8004` 都正常。
2. 队友确认能通过 VPN 或其他方式访问 `http://你的校内IP:8003/`。
3. 你确认 `STATIC_BASE_URL` 已经不是 `localhost`。
4. 你确认前端 `.env.local` 的 `VITE_API_BASE_URL` 是 `http://你的校内IP:8003`。
5. 先测文字对话。
6. 再测 `engine=gpt_sovits`。
7. 最后接入 CosyVoice 队友的地址，测 `engine=cosyvoice`。

按这个顺序排查，能最快区分是网络、CORS、LLM、GPT-SoVITS 还是 CosyVoice 的问题。
