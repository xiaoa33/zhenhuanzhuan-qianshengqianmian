# 《甄嬛传·千声千面》前端设计文档

---

## 一、项目概述

**系统名称**：甄嬛传·千声千面

**系统功能**：用户选择身份（现代人或剧中角色），与《甄嬛传》经典角色进行AI对话。系统结合角色画像由LLM生成回复，分析情绪后驱动声音合成模型输出语音，同步呈现文字气泡与数字人说话效果。

---

## 二、技术栈

- **框架**：React
- **样式**：CSS（含 CSS Variables、Keyframes动画）
- **截图导出**：html2canvas
- **字体**：Ma Shan Zheng（毛笔风，Google Fonts）+ 宋体（正文）

## 五、页面详细设计

### 5.1 主页（HomePage）

**背景**
- 全屏背景图：`/yiping-frontend/resource/背景图/主页背景图.jpg`
- 图片上叠加半透明深色遮罩：`rgba(0,0,0,0.5)`
- 图片本身做 **Ken Burns 效果**：缓慢放大（`scale: 1→1.08`），持续 15 秒循环，营造电影感

```css
.home-bg {
  animation: kenBurns 15s ease-in-out infinite alternate;
}
@keyframes kenBurns {
  from { transform: scale(1); }
  to   { transform: scale(1.08); }
}
```

**中央内容（垂直居中）**
- 系统标题：`甄嬛传·千声千面`，Ma Shan Zheng 字体，金色 `#c9a84c`，字号 `4rem`，带轻微文字阴影
- 副标题：`入宫廷，闻千声`，宋体，米白色，字号 `1.1rem`，字间距 `0.3em`
- 进入按钮：`进入宫廷`，金色边框，半透明深色背景，悬停时背景变金色、文字变深色，过渡 `0.3s`
- 音乐控制图标：右下角固定悬浮，♪ 图标，点击切换静音/播放状态

**背景音乐**
- 音频文件：`/yiping-frontend/resource/长相思 (甄嬛传背景音乐).mp3`
- 页面加载后自动播放，音量 `0.4`，循环播放（`loop`）
- 全局持续播放，切换页面不中断

**入场动画**

页面加载时，角色卡片从下方依次飞入，每张间隔 `100ms`：

```css
@keyframes cardEnter {
  from { opacity: 0; transform: translateY(40px); }
  to   { opacity: 1; transform: translateY(0); }
}
.character-card:nth-child(n) {
  animation: cardEnter 0.4s ease forwards;
  animation-delay: calc(n * 100ms);
}
```

**悬停效果**
- `transform: scale(1.05)`
- 边框变亮金色 `#e8c97a`
- `transition: 0.2s`

**点击效果**

#### 5.3.1 左侧：数字人区

- 显示当前对话角色的剧照（或数字人视频，若后端已返回）

- 角色名显示在图片下方，金色，Ma Shan Zheng，字号 `1.4rem`

```css

  display: inline-block;
  width: 4px;
  height: 16px;
  background: #c9a84c;
  margin: 0 3px;
  animation: wave 0.8s ease-in-out infinite;
```

### 5.4 对话总结卡片（SummaryCard）

**触发方式**：用户点击"结束对话"后，页面中央弹出模态弹窗。

**弹窗样式**
- 居中弹窗，宽度 `420px`
- 背景：`rgba(20,15,10,0.92)`，金色边框，`backdrop-filter: blur(12px)`，圆角 `16px`
- 出现动画：从 `scale(0.85)` + `opacity 0` 到 `scale(1)` + `opacity 1`，`0.3s`

**内容布局（从上到下）**

1. 顶部标题：`今日宫廷密录`，Ma Shan Zheng，金色，居中
2. 装饰分隔线：金色细线
3. 角色头像（`60px` 圆形）+ 角色名
4. 对话统计：`共交谈 N 句`，米白色小字
5. 角色态度评价（由后端 LLM 返回）：如 `华妃娘娘今日对你：戒备有加`，金色高亮
6. 角色风格总结语（由后端 LLM 返回）：如 `你尚未入她眼，需再努力`，斜体，米白色
7. 装饰分隔线
8. 底部两个并排按钮：
   - `保存图片`：调用 `html2canvas` 截取卡片内容，导出为 PNG 图片下载
   - `再聊一次`：关闭弹窗，清空对话记录，重新开始

**后端需配合提供的字段（对接接口时参考）**

```json
{
  "summary": {
    "attitude": "戒备有加",
    "comment": "你尚未入她眼，需再努力",

  }
```

## 六、数字人集成说明

**方案**：使用 SadTalker 或 MuseTalk，由负责模型的队友输入角色剧照 + 合成音频，输出说话视频。

**前端对接方式**

- 后端返回音频 URL 和视频 URL
- 音频：`<audio>` 标签 `autoplay` 播放
- 视频：`<video>` 标签替换左侧剧照展示，播放完毕后自动恢复剧照静态图
- 等待期间（视频生成中）：左侧显示角色静态剧照 + 底部加载指示动画（金色旋转光圈）

**降级方案**：若数字人视频生成延迟较高，汇报演示时使用预录制好的精彩对话视频片段播放，而非实时生成。
