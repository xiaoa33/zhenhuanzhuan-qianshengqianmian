# 甄嬛传·千声千面 — 后端设计文档

## 1. 整体架构

```
前端 (React, :5173)
    │
    ├── POST /asr            ──┐
    ├── POST /chat           ──┤
    ├── POST /synthesize     ──┤──→ yiping-backend (FastAPI, :8000)
    ├── POST /digital-human  ──┤
    └── POST /summary        ──┘
                                         │
                    ┌────────────────────┼────────────────────┐
                    ▼                    ▼                    ▼
         [模块A] xiao-asr_llm    [模块B] TTS语音合成     [本地] SadTalker
         POST :8001/asr
         POST :8001/generate     POST :8002/synthesize   subprocess调用
         POST :8001/summarize
```

**职责划分**：

| 模块 | 负责内容 |
|------|---------|
| `yiping-backend` | HTTP接口、请求路由、静态文件托管、SadTalker调用 |
| `xiao-asr_llm`（模块A） | ASR语音识别、LLM角色对话生成、情绪分析、对话总结 |
| TTS语音合成（模块B） | 角色声音合成，输出音频文件 |

---

## 2. yiping-backend API 接口规约

所有接口：`Content-Type: application/json`，Base URL：`http://localhost:8000`

### POST /asr

**功能**：用户语音识别，返回识别文本

**请求体**：`multipart/form-data`，字段名 `audio`

**响应体**：
```json
{
  "text": "娘娘今日心情如何？",
  "language": "zh",
  "confidence": 0.93
}
```

### POST /chat

**功能**：角色对话，返回角色回复文本和情绪标签

**请求体**：
```json
{
  "character_id": "huafei",
  "user_identity": "modern",
  "user_role_id": null,
  "user_role_name": null,
  "history": [
    { "role": "user", "text": "华妃娘娘好" },
    { "role": "character", "text": "贱人就是矫情" }
  ],
  "user_input": "娘娘今日心情如何？"
}
```

**响应体**：
```json
{
  "text": "本宫今日心情极佳，你倒是有眼力见儿。",
  "emotion": "喜悦",
  "tts_texts": {
    "cosyvoice": "[laughter]本宫今日心情极佳，你倒是有眼力见儿。",
    "gpt_sovits": "本宫今日心情极佳，你倒是有眼力见儿。"
  }
}
```

`emotion` 枚举值：`愤怒` | `悲伤` | `喜悦` | `平静`

---

### POST /synthesize

**功能**：语音合成，返回音频文件 URL

**请求体**：
```json
{
  "character_id": "huafei",
  "text": "本宫今日心情极佳，你倒是有眼力见儿。",
  "emotion": "喜悦"
}
```

**响应体**：
```json
{
  "audio_url": "http://localhost:8000/static/audio/reply_abc123.wav"
}
```

---

### POST /digital-human

**功能**：驱动数字人生成说话视频，返回视频 URL

**请求体**：
```json
{
  "character_id": "huafei",
  "audio_url": "http://localhost:8000/static/audio/reply_abc123.wav"
}
```

**响应体**：
```json
{
  "video_url": "http://localhost:8000/static/video/reply_abc123.mp4"
}
```

失败时返回 `{ "video_url": null }`，前端自动降级显示静态剧照。

---

### POST /summary

**功能**：生成对话总结评价

**请求体**：
```json
{
  "character_id": "huafei",
  "messages": [
    { "role": "user", "text": "华妃娘娘好" },
    { "role": "character", "text": "贱人就是矫情" }
  ]
}
```

**响应体**：
```json
{
  "attitude": "戒备有加",
  "comment": "你尚未入她眼，需再努力",
  "rounds": 8
}
```

`rounds` 由后端统计 `messages` 中 `role == "user"` 的数量。

## 3. 外部模块A接口定义（ASR + LLM文字生成 + 情绪分析）

**模块**：`xiao-asr_llm`  
**服务地址**：`http://localhost:8001`（通过环境变量 `LLM_SERVICE_URL` 配置）

### POST /generate — 角色对话生成

**请求体**：
```json
{
  "character_id": "huafei",
  "user_identity": "modern",
  "user_role_id": null,
  "user_role_name": null,
  "history": [
    { "role": "user", "text": "华妃娘娘好" },
    { "role": "character", "text": "贱人就是矫情" }
  ],
  "user_input": "娘娘今日心情如何？"
}
```

**响应体**：
```json
{
  "text": "本宫今日心情极佳，你倒是有眼力见儿。",
  "emotion": "喜悦"
}
```

**字段说明**：
- `character_id`：角色唯一标识，用于加载对应角色的人设 prompt
- `user_identity`：`modern`（现代来客）或 `ancient`（宫廷中人）
- `user_role_id` / `user_role_name`：用户扮演的角色，`ancient` 身份时非空，供 LLM 调整称呼和口吻
- `history`：最近的对话历史（建议保留最近 10 轮）
- `emotion`：LLM 根据回复内容分析并返回，4选1枚举值

---

### POST /summarize — 对话总结生成

**请求体**：
```json
{
  "character_id": "huafei",
  "messages": [
    { "role": "user", "text": "华妃娘娘好" },
    { "role": "character", "text": "贱人就是矫情" }
  ]
}
```

**响应体**：
```json
{
  "attitude": "戒备有加",
  "comment": "你尚未入她眼，需再努力"
}
```

**字段说明**：
- `attitude`：角色对用户的整体态度，4字以内，供展示在总结卡片醒目位置
- `comment`：LLM 生成的风格化总结语，20字以内，符合古风口吻

---

## 4. 外部模块B接口定义（语音合成 TTS）

**模块**：待命名（TTS语音合成）  
**服务地址**：`http://localhost:8002`（通过环境变量 `TTS_SERVICE_URL` 配置）

### POST /synthesize — 角色语音合成

**请求体**：
```json
{
  "character_id": "huafei",
  "text": "本宫今日心情极佳，你倒是有眼力见儿。",
  "emotion": "喜悦"
}
```

**响应体（方案A，推荐）**：
```json
{
  "audio_path": "/absolute/path/to/output.wav"
}
```
> `yiping-backend` 读取该路径，将文件复制/链接到 `static/audio/` 后返回 HTTP URL 给前端。

**响应体（方案B，备选）**：
```json
{
  "audio_base64": "UklGRiQ...",
  "format": "wav"
}
```
> `yiping-backend` 解码 base64，保存为文件后返回 HTTP URL。

**字段说明**：
- `character_id`：用于选择对应角色的声音模型
- `emotion`：用于调整语调、语速、音色等情感参数（如模型支持）

## 5. 数字人实现（SadTalker）

### 调用流程

```
POST /digital-human { character_id, audio_url }
    ↓
1. 将 audio_url 转为本地路径
   e.g. http://localhost:8000/static/audio/xxx.wav → ./static/audio/xxx.wav
    ↓
2. 读取角色参考图
   ./resource/portraits/{character_id}.jpg
    ↓
3. 调用 SadTalker subprocess
   python {SADTALKER_PATH}/inference.py
       --driven_audio  ./static/audio/xxx.wav
       --source_image  ./resource/portraits/{character_id}.jpg
       --result_dir    ./static/video/
       --still                 # 减少头部晃动
       --preprocess full
    ↓
4. 扫描 result_dir 取最新生成的 .mp4
    ↓
返回: { "video_url": "http://localhost:8000/static/video/xxx.mp4" }
```

### SadTalker 安装（Windows）

```powershell
git clone https://github.com/OpenTalker/SadTalker.git
cd SadTalker\SadTalker
uv venv --python 3.8 .venv
.venv\Scripts\activate
# lmdb 2.x 与 Python 3.8 C API 不兼容，需先装预编译旧版
uv pip install "lmdb==1.4.1" --only-binary :all:
uv pip install -r requirements.txt
```

| GFPGAN 离线补丁 gfpgan/ | https://pan.baidu.com/s/1P4fRgk9gaSutZnn8YW034Q 密码：`sadt` | 同上 |

## 6. Mock 数据策略

外部模块未就绪时，通过环境变量 `USE_MOCK=true` 启用 Mock 模式：

| 接口 | Mock 内容 | 实际替换位置 |
|------|---------|------------|
| `/chat` | 每角色预设5条经典台词轮流返回，emotion 随机 | `services/llm_client.py` → `call_generate()` |
| `/synthesize` | 返回 `static/audio/mock_silence.wav` URL | `services/tts_client.py` → `call_synthesize()` |
| `/summary` | 返回固定 attitude/comment 模板 | `services/llm_client.py` → `call_summarize()` |
| `/digital-human` | 直接调用 SadTalker，不走 mock | — |

所有 mock 预设数据集中在 `mock/mock_data.py`。

## 7. 项目文件结构

```
yiping-backend/
├── main.py                          # FastAPI 应用入口，CORS，路由注册，静态文件挂载
├── routers/
│   ├── chat.py                      # POST /chat
│   ├── synthesize.py                # POST /synthesize
│   ├── digital_human.py             # POST /digital-human
│   └── summary.py                   # POST /summary
├── services/
│   ├── llm_client.py                # 调用模块A（httpx异步）
│   └── tts_client.py                # 调用模块B（httpx异步）
├── digital_human/
│   └── sadtalker.py                 # SadTalker subprocess 封装
├── mock/
│   └── mock_data.py                 # 各角色预设回复、情绪、总结数据
├── static/
│   ├── audio/                       # 生成的音频文件
│   │   └── mock_silence.wav         # mock用静音音频
│   └── video/                       # SadTalker生成的视频文件
├── resource/
│   └── portraits/                   # 12个角色参考图（character_id.jpg）
├── .env.example                     # 环境变量模板
├── requirements.txt                 # Python依赖
└── README.md                        # 启动说明 + mock替换指引
```

## 8. 环境变量配置

```env
# .env.example

# 是否使用 mock 数据（true = mock，false = 调用真实外部模块）
USE_MOCK=true

# 外部模块A：LLM文字生成 + 情绪分析
LLM_SERVICE_URL=http://localhost:8001

# 外部模块B：TTS语音合成
TTS_SERVICE_URL=http://localhost:8002

# SadTalker 仓库路径（数字人功能必填）
SADTALKER_PATH=../SadTalker/SadTalker

# SadTalker Python 3.8 虚拟环境解释器路径（Windows 必填，避免与主项目 Python 3.13 冲突）
SADTALKER_PYTHON=<SadTalker项目路径>/.venv/Scripts/python.exe

# 对外暴露的静态资源基础 URL（用于生成 audio_url / video_url）
STATIC_BASE_URL=http://localhost:8000

# 后端监听端口
PORT=8000
```

## 9. 技术选型

| 项目 | 选型 | 原因 |
|------|------|------|
| Web框架 | FastAPI | 原生异步、自动生成 Swagger 文档 |
| HTTP客户端 | httpx | 支持 async/await，适配 FastAPI |
| 数字人 | SadTalker | 开源、单GPU可运行、效果稳定 |
| CORS | fastapi.middleware.cors | 允许前端 :5173 跨域 |
| 静态文件 | fastapi.staticfiles | 托管生成的音频/视频文件 |

## 10. 启动与验证

```bash
cd yiping-backend
pip install -r requirements.txt
cp .env.example .env
# 按需编辑 .env

uvicorn main:app --reload --port 8000
```

**验证步骤**：
1. 访问 `http://localhost:8000/docs` → 查看自动生成的 Swagger 接口文档
2. 启动前端 `npm run dev`，访问 `http://localhost:5173`
3. 选择角色，发送消息 → 浏览器 Network 面板检查4个接口的请求/响应
4. Mock 模式下：文字回复正常；音频 URL 可访问（静音）；数字人视频生成后显示在左侧
