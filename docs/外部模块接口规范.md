# 外部模块接口规范

本文档面向负责 **LLM 文字生成模块（模块A）** 和 **TTS 语音合成模块（模块B）** 的队友。

`yiping-backend` 通过 HTTP 请求调用这两个模块。你们只需要各自跑一个本地 HTTP 服务，监听指定端口，实现下面定义的接口即可。

---

## 整体调用关系

```
前端 (:5173)
    │  POST /chat / /synthesize / /summary
    ▼
yiping-backend (:8000)          ← 你们不用管这里
    │
    ├── POST http://localhost:8001/generate    → 模块A（LLM）
    ├── POST http://localhost:8001/summarize   → 模块A（LLM）
    └── POST http://localhost:8002/synthesize  → 模块B（TTS）
```

`yiping-backend` 默认处于 `USE_MOCK=true` 状态，**你们的模块没完成之前主后端照常运行**，不互相阻塞。模块就绪后，主后端将 `.env` 里的 `USE_MOCK` 改为 `false` 即可切换到真实调用。

---

## 模块A：LLM 文字生成（端口 8001）

### 接口一：POST /generate — 角色对话生成

**请求体**

```json
{
  "character_id": "huafei",
  "user_identity": "modern",
  "user_role_id": null,
  "user_role_name": null,
  "history": [
    { "role": "user", "text": "华妃娘娘好" },
    { "role": "character", "text": "贱人就是矫情" }
  ],
  "user_input": "娘娘今日心情如何？"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `character_id` | string | 角色唯一标识，用于加载对应角色人设 prompt |
| `user_identity` | string | `"modern"`（现代来客）或 `"ancient"`（宫廷中人） |
| `user_role_id` | string \| null | 用户扮演的角色 ID，`ancient` 时非空 |
| `user_role_name` | string \| null | 用户扮演的角色名，`ancient` 时非空，供 LLM 调整称呼 |
| `history` | array | 最近对话历史，建议保留最近 10 轮 |
| `user_input` | string | 用户本次输入 |

**响应体**

```json
{
  "text": "本宫今日心情极佳，你倒是有眼力见儿。",
  "emotion": "喜悦"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `text` | string | 角色回复的文本内容 |
| `emotion` | string | LLM 根据回复内容判断的情绪标签，见下方枚举值 |

**emotion 枚举值**

> 当前定义了 4 种情绪标签，具体分类可与模块B队友对齐——TTS 会用这个字段选择对应的语调合成音频。如果 TTS 那边支持更细的情绪粒度，可以在此基础上扩展，但要保持双方一致。

| 值 | 含义 |
|----|------|
| `喜悦` | 开心、愉快、满意 |
| `愤怒` | 生气、不满、威胁 |
| `悲伤` | 难过、哀怨、委屈 |
| `平静` | 无明显情绪、陈述事实 |

---

### 接口二：POST /summarize — 对话总结生成

**请求体**

```json
{
  "character_id": "huafei",
  "messages": [
    { "role": "user", "text": "华妃娘娘好" },
    { "role": "character", "text": "贱人就是矫情" }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `character_id` | string | 角色唯一标识 |
| `messages` | array | 完整对话记录，格式同 `history` |

**响应体**

```json
{
  "attitude": "戒备有加",
  "comment": "你尚未入她眼，需再努力"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `attitude` | string | 角色对用户的整体态度，**4字以内**，展示在总结卡片醒目位置 |
| `comment` | string | 风格化总结语，**20字以内**，符合古风口吻 |

## 模块B：TTS 语音合成（端口 8002）

### 接口：POST /synthesize — 角色语音合成

**请求体**

```json
{
  "character_id": "huafei",
  "text": "本宫今日心情极佳，你倒是有眼力见儿。",
  "emotion": "喜悦"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `character_id` | string | 角色唯一标识，用于选择对应角色的声音模型 |
| `text` | string | 需要合成的文本 |
| `emotion` | string | 情绪标签（同上方枚举值），用于调整语调/语速/音色 |

**响应体（推荐方案 A：返回文件路径）**

```json
{
  "audio_path": "/absolute/path/to/output.wav"
}
```

`yiping-backend` 会读取该路径，将文件复制到 `static/audio/` 后对外提供 HTTP URL。**路径必须是绝对路径**，且对 `yiping-backend` 进程可读。

**响应体（备选方案 B：返回 base64）**

```json
{
  "audio_base64": "UklGRiQ...",
  "format": "wav"
}
```

`yiping-backend` 会自动识别这两种格式，任选其一即可。

## 如何监听端口：FastAPI 示例

以下是最简单的骨架代码，可以直接作为起点。

```python

    user_role_name: Optional[str] = None
    history: list
    user_input: str

class SummarizeRequest(BaseModel):
    character_id: str
    messages: list

@app.post("/generate")
def generate(req: GenerateRequest):
    # TODO: 调用你们的 LLM，生成角色回复和情绪标签
    text = "..."
    emotion = "平静"  # 从 ["喜悦", "愤怒", "悲伤", "平静"] 中选一个
    return {"text": text, "emotion": emotion}

@app.post("/summarize")
def summarize(req: SummarizeRequest):
    # TODO: 调用你们的 LLM，生成总结
    return {"attitude": "戒备有加", "comment": "你尚未入她眼，需再努力"}
```

```bash
# 启动（在模块A目录下）

```

## 本地同时运行三个服务

三个服务各自是独立进程，只要端口不冲突就能在同一台机器上同时跑。打开三个终端窗口分别启动：

```bash
# 终端1：主后端
cd yiping-backend
uvicorn main:app --reload --port 8000
```

```bash
# 终端2：LLM 模块（模块A）
cd xiao-asr_llm
uvicorn main:app --reload --port 8001
```

```bash
# 终端3：TTS 模块（模块B）
cd <你们的TTS目录>
uvicorn main:app --reload --port 8002
```

> VS Code 左侧面板支持同时开多个终端，不需要多开窗口。

## 与 yiping-backend 对接步骤

模块就绪后，通知yiping（主后端）做以下操作：

1. 编辑 `yiping-backend/.env`：
   ```env
   USE_MOCK=false
   LLM_SERVICE_URL=http://localhost:8001   # 模块A地址，默认已是这个
   TTS_SERVICE_URL=http://localhost:8002   # 模块B地址，默认已是这个
   ```

2. 确认你们的服务正在运行（可以用 `curl` 或浏览器访问各自的 `/docs` 验证）

3. 重启 `yiping-backend`（`uvicorn main:app --reload --port 8000`）

对接的具体代码在 `yiping-backend/services/llm_client.py` 和 `tts_client.py`，如果你们的字段名有差异，在这两个文件里改就行，主后端其他代码不用动。
