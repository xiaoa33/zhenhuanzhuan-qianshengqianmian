# 手动启动指令

## 端口约定

| 服务 | 端口 |
|---|---|
| xiao-asr_llm | `8001` |
| CosyVoice 本地隧道 | `8002` |
| yiping-backend | `8003` |
| GPT-SoVITS | `8004` |
| yiping-frontend | `5173` |

## 0. 确认前端后端地址

如果在服务器本机浏览器访问，可用：

```env
VITE_API_BASE_URL=http://localhost:8003
VITE_TTS_ENGINE=cosyvoice
```

如果在自己电脑浏览器访问服务器前端，把 `localhost` 改成服务器 IP，例如：

```env
VITE_API_BASE_URL=http://10.160.4.184:8003
VITE_TTS_ENGINE=cosyvoice
```

配置文件位置：

```text
yiping-frontend/.env.local
```

修改 `.env.local` 后必须重启前端。

## 1. 连接 CosyVoice 隧道

终端 1：

```bash
ssh -N -L 8002:localhost:8003 -p 40916 root@connect.bjb1.seetacloud.com
```

这个终端不要关。它表示：

```text
本机 localhost:8002 -> AutoDL localhost:8003
```

另开终端验证：

```bash
curl http://localhost:8002/
```

或：

```bash
curl http://localhost:8002/docs
```

## 2. 启动 xiao-asr_llm

终端 2：

```bash
cd xiao-asr_llm
python -m uvicorn main:app --host 0.0.0.0 --port 8001
```

验证：

```bash
curl http://localhost:8001/
```

## 3. 启动 GPT-SoVITS

终端 3：

```bash
cd xiao-gpt-sovits

GSV_BACKEND=persistent GSV_CACHE_SIZE=1 \
  python -m uvicorn main:app --host 0.0.0.0 --port 8004
```

修改绝对路径为相对路径之后的临时启动指令：
```bash
cd xiao-gpt-sovits

GSV_ROOT=../../GPT-SoVITS \
GSV_LIST_DIR="../../dataset/gpt_sovits_lists/by_role" \
GSV_BACKEND=persistent \
GSV_CACHE_SIZE=1 \
python -m uvicorn main:app --host 0.0.0.0 --port 8004
```

验证：

```bash
curl http://localhost:8004/
```

## 4. 启动 yiping-backend

终端 4：

```bash
cd yiping-backend
python -m uvicorn main:app --host 0.0.0.0 --port 8003
```

验证：

```bash
curl http://localhost:8003/
```

## 5. 启动 yiping-frontend

终端 5：

```bash
cd yiping-frontend
npm run dev -- --host 0.0.0.0 --port 5173 --strictPort
```

本机访问：

```text
http://localhost:5173
```

远程访问示例：

```text
http://10.160.4.184:5173
```

## 6. 常用联通测试

测试主后端调用 GPT-SoVITS：

```bash
curl -X POST http://localhost:8003/synthesize \
  -H "Content-Type: application/json" \
  -d '{"character_id":"huafei","text":"本宫今日心情尚可。","emotion":"平静","engine":"gpt_sovits"}'
```

测试主后端调用 CosyVoice 流式：

```bash
curl -N -X POST http://localhost:8003/synthesize/stream \
  -H "Content-Type: application/json" \
  -d '{"character_id":"huafei","text":"本宫今日心情尚可。","emotion":"平静","engine":"cosyvoice"}'
```

测试即兴对话：

```bash
curl -N -X POST http://localhost:8003/duet/start \
  -H "Content-Type: application/json" \
  -d '{"character_a":"huafei","character_b":"zhenhuan","context":"御花园偶遇","starter":"huafei","max_rounds":2}'
```

## 7. 常见问题

如果提示端口被占用，先找出占用进程：

```bash
lsof -nP -iTCP:8001 -sTCP:LISTEN
lsof -nP -iTCP:8002 -sTCP:LISTEN
lsof -nP -iTCP:8003 -sTCP:LISTEN
lsof -nP -iTCP:8004 -sTCP:LISTEN
lsof -nP -iTCP:5173 -sTCP:LISTEN
```

确认是旧项目进程后，优先停止对应终端。若终端找不到，可用 PID 结束：

```bash
kill <PID>
```

如果进程没有退出，再用：

```bash
kill -9 <PID>
```

例如 `lsof` 显示：

```text
COMMAND   PID       USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
python  12345 wangxinran   12u  IPv4  ...       0t0  TCP *:8003 (LISTEN)
```

则终止：

```bash
kill 12345
```

确认端口已释放：

```bash
lsof -nP -iTCP:8003 -sTCP:LISTEN
```

也可以按项目常用端口逐个清理。先查看：

```bash
for port in 5173 5174 8001 8002 8003 8004 18002; do
  echo "== port ${port} =="
  lsof -nP -iTCP:${port} -sTCP:LISTEN || true
done
```

确认都是本项目旧进程后，再执行：

```bash
for port in 5173 5174 8001 8002 8003 8004 18002; do
  pids=$(lsof -tiTCP:${port} -sTCP:LISTEN)
  if [ -n "$pids" ]; then
    echo "Killing port ${port}: ${pids}"
    kill $pids
  fi
done
```

如果仍有端口未释放，再查看并手动 `kill -9 <PID>`。

如果即兴对话没有输出，优先看启动 `yiping-backend` 的终端日志。正常应出现：

```text
duet start
duet turn
duet LLM ok
duet TTS first audio
```

如果只有 `duet LLM ok`，没有 `duet TTS first audio`，通常是 CosyVoice 流式接口或 SSH 隧道有问题。
